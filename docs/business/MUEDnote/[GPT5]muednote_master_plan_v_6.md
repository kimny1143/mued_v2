# 1. エグゼクティブサマリー

MUEDnote は、音楽制作者（DAWユーザー）のための **「認知負荷の外部化インフラ」**であり、
**制作中のフロー状態を一切途切れさせないまま、乱文（思考断片）を記録・構造化し、知的資産に変換するシステム** である。

本統合設計の中心方針:
- 表のUXは "完全沈黙" — 入力バーだけが一瞬現れ、返事はしない。
- 裏では AI が常時稼働し、要約・タグ付け・拡張・HLA処理を全自動で行う。
- MVP は DAW 連携ゼロで出す（最速で価値提供）。
- 製品版では AU/VST/AAX による完全DAW統合を実現する。
- 市場の中心は B2C。B2B はログ解析を利用した派生価値で展開する。

本書は MUEDnote の哲学、UX、AI構造、技術、事業、成長戦略すべてを一気通貫で記述する最終仕様である。

# 2. プロダクト哲学：制作行動の身体性を絶対に壊してはならない

MUEDnote の存在理由は「音楽制作における思考の蒸発を防ぐ」ことであり、
AI と会話するツールではなく **“考えた瞬間に捨てられる場所”** を提供することである。

## 2.1 中核哲学
1. **制作フローへの割り込み禁止**  
   DAW の操作 → メモ → 再開。この一連の切り替えは、集中の断絶を生む。MUEDnote はこれを根絶する。

2. **乱文は宝である**  
   制作中に吐き捨てられる断片的な思考こそが“暗黙知”の本体である。それを即座に掬い上げ、後から使える形にする。

3. **AIは沈黙するが、裏では働く**  
   表では一切喋らず、裏で解析と拡張を続ける “Silent AI Architecture”。

4. **外部脳としての永続性**  
   一度 MUEDnote に思考を預け始めると、もう離れられない。「脳の拡張メモリ」として機能する。

# 3. UX憲法：0.5秒・沈黙・ワンライン

制作現場の身体性に寄り添うための UX 原則。

## 3.1 0.5秒の壁
- **ホットキー押下 → 入力バー出現 → 入力開始が0.5秒以内**
- UIは DAW の前に即時オーバーレイされる
- Enter または Esc で 0.1秒以内に瞬時に消滅
- DAWフォーカスへ自動復帰

## 3.2 沈黙のAI（Silent Console）
- 入力後、AIは **何も返さない**
- 返事・助言・相槌すら禁止
- 視覚フィードバックは “微細な点滅ドット” のみ
- ただし裏で以下は実行：
  - 乱文解析
  - 文脈推定
  - HLA処理
  - 要約の蓄積
  - タイムライン統合
  - 自動メタデータ生成

## 3.3 入力UI（DRAINPIPE UX）
- グローバルホットキー：例 `Cmd + Shift + M`
- 1行テキストバーのみ
- 入力した瞬間に DAW の世界へ戻れる身体感覚を提供

# 4. HLA：Human Learning Algorithm（人間学習アルゴリズム）

v3.2 の完全版を中心に、裏側自動処理として統合する。

## 4.1 HLA の目的
人間の制作行動には、明文化されていない「癖・意図・判断基準・ストレスパターン」が存在する。  
HLA はそれを解析し、以下に変換する：

- 意図（何をしようとしていたか）
- 行動の特徴量（口癖、専門語反復、負荷指標）
- 感情傾向（ネガティブスパイラル検知）
- DAWの文脈（プロジェクトID、時間帯など）
- ラベル（#Mix、#Melody、#Fix3kHz など）
- スランプ予兆
- 成功パターンの抽出

## 4.2 ステージ構成
### Stage 1：Intent Recognition
乱文から**対象・行動・修飾**を抽出し、最初のタグへ変換。

### Stage 2：Feature Extraction
言語パターン、時間帯、作業密度などを特徴量化。

### Stage 3：Label Generation
クラスタリングにより経験則ラベル（WeakPoint/StrongPointなど）を生成。

これらはすべて裏で動き、UIには一切出ない。

# 5. 技術アーキテクチャ（Silent AI Architecture）

MUEDnote は「UIレイヤ」「Agent」「Daemon」「AIエンジン」の4層構造で動作する。

## 5.1 MUEDnote Agent（デスクトップ常駐）
- Rust/Tauri による高速UI
- 0.5秒の壁の守護者
- 乱文 → ローカルキュー保存
- DAWフォーカス管理

## 5.2 Daemon（バックグラウンド）
- Node/Python 等で動く非同期処理
- ログ ingestion
- AIへの送信
- HLA処理
- Firestore / Supabase などのデータ永続化

## 5.3 AIエンジン
- GPT-4o mini・Gemini Flash など高速モデル
- Function / JSON mode による構造化

## 5.4 データ構造
- Raw Log（乱文）
- Structured Log（HLAによる構造）
- Summary Log（AI要約）
- Contextual Metadata（DAW情報）
- Pattern DB（個人特徴）

# 6. DAW連携戦略（MVP→完全版）

## 6.1 Phase 1：MVP（DAW連携なし）
- 乱文の価値だけで勝負
- “外部脳”としての習慣形成が目的

## 6.2 Phase 2：製品版（DAW完全対応）
以下をAU/VST/AAXで実現：

### Protocol 1：Window Title / Active Process Monitoring
プロジェクト名を自動取得

### Protocol 2：File Path Tracking
プロジェクトファイル更新を監視

### Protocol 3：MIDI / OSC（任意）
再生/停止・アクティブトラック状況の間接取得

### Protocol 4：Keystroke Context Linking
直前操作の推定（Cmd+Z → Undo系ログなど）

この全てをプラグインで統合し、世界初の**「DAWの内側に溶け込む外部脳」**を実現する。
