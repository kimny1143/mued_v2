## 管理ルール

### 1. ファイル命名規則

```
YYYY-MM-DD_<内容>.md
```

例:
- `2025-12-22_ios-device-test.md`
- `2025-12-23_testflight-setup.md`

### 2. 同日の追加作業

同じ日に追加作業があった場合は、**既存ファイルに追記**する（新規ファイルを作らない）。

```markdown
## 追加テスト: <タイトル>（同日午後）

### 内容
...
```

### 3. CHANGELOG.md の更新タイミング

フェーズ完了時に更新:
- Phase 1 完了 → CHANGELOG に Phase 1 セクション追加
- Phase 2 完了 → CHANGELOG に Phase 2 セクション追加

日次の詳細は `YYYY-MM-DD_*.md` に記載し、CHANGELOG はサマリーのみ。

### 4. PoC との区別

| 種別 | 場所 | 内容 |
|------|------|------|
| PoC | `../poc-log.md` | 技術検証（実装可否の判断） |
| MVP | このフォルダ | 実装（プロダクト開発） |
