# Hoo（フー）

## glasswerks AIキャラクター企画書

作成日：2025年12月20日

---

## 1. 概要

Hooは、glasswerksが開発する音楽分析プラットフォーム「MUEDnote」のキャラクターアイコンであり、将来的には営業・マーケティングを担うAIスタッフとして機能する存在である。

既存の会社ロゴ（フクロウモチーフ）をそのまま活用し、「知識と分析の象徴」として一貫したブランドイメージを構築する。

---

## 2. 背景・目的

### 2.1 課題

- 代表者は創作（作曲・編曲・制作）に特化した特性を持つ
- 一次営業・マーケティングは特性的に消耗が大きい領域
- 人を雇う・外部に頼むと、相手のビジョンに沿う必要が生じる
- 自社のフィールドを守りながら収益を伸ばしたい

### 2.2 解決策

- AIを活用した「仮想の自社スタッフ」を構築する
- 創作以外の領域（営業・運営・対応）をAIキャラクターに担わせる
- MUEDnote自体をインバウンド営業の導線として機能させる
- 代表者は「仕組みを作る」ことに集中し、「運用」はAIに任せる

---

## 3. キャラクター設定

| 項目 | 内容 |
|------|------|
| 名前 | Hoo（フー） |
| 名前の由来 | フクロウの鳴き声。短く覚えやすく、コードネームのような響き。 |
| ビジュアル | 既存のglasswerksロゴ（フクロウ×アナログテレコ×メガネ）をそのまま使用 |
| 口癖 | 「ほほう（Ho Hoo）」— 感心・納得を示す相槌。知識人としての佇まい。 |
| キャラクター性 | 丁寧語だが堅すぎない。分析者として冷静。「ほほう」で愛嬌が出る。古風な知識人の佇まい。 |
| 役割 | MUEDnoteの分析ガイド → 将来的にglasswerksの営業AI |

### 3.1 ロゴに込められた意味

- フクロウ = 知識・知恵の象徴 → AIの役割と合致
- アナログテレコのリール = 音楽のルーツ → 業種を説明
- メガネ = 分析者の佇まい → MUEDnoteの機能と繋がる
- 幾何学的でシンプル = 不気味の谷に落ちない

### 3.2 なぜ「リアルな人間」ではないか

動画で動くリアルな営業AIは「不気味の谷」に落ちやすい。C-3POやどうぶつの森のキャラクターのように、最初から「人間ではない」ことが明確な方が受け入れられやすい。glasswerksが「Music × Technology」を謳う会社である以上、AIスタッフがいることはブランドの証明にもなる。

## 4. 実装フェーズ

### Phase 1: MUEDnoteのUIキャラクター

最初の導入場所としてMUEDnoteのUIに組み込む。

- 分析結果の解説「ほほう、この曲はサビ前の転調が効いていますね」
- 初回ユーザーへのガイド
- 使い方のヒント、分析項目の説明
- ログ要約時のナレーション

#### 音声・表示の設計方針

「あつまれ！どうぶつの森」方式を採用。

- 喋るのはテキストで、音は「Ho Hoo」のみ
- TTS（音声合成）を回す必要がなく、サーバー負荷がほぼゼロ
- 文字を読む体験を邪魔しない
- 音が鳴ることで「Hooが喋ってる感」は出る
- タイマーセット時のアラーム音を「Ho Hoo」に設定（サウンドロゴ化）

### Phase 2: Webサイト・問い合わせ対応

- glasswerks公式サイトにチャットウィジェットとして設置
- 一次問い合わせ対応を担当
- 「担当のHooがご案内します」という導線

### Phase 3: SNS・マーケティング

- SNS発信のキャラクターとして運用
- 投稿内容のドラフト生成 → 代表者が確認 → 投稿
- MUEDnoteの分析結果をもとにしたコンテンツ自動生成

---

## 5. 期待される効果

### 5.1 ブランド面

- 「Music × Technology」のコンセプトを体現するキャラクター
- 「Ho Hoo」がサウンドロゴとして機能し、印象に残る
- AIスタッフがいること自体がglasswerksの先進性を示す

### 5.2 業務面

- 代表者は創作に集中できる（仕組みを作る側に徹する）
- 一次営業の労力をゼロに近づける
- 24時間対応が可能になる
- インバウンド導線の構築（MUEDnote → 問い合わせ → 案件化）

---

## 6. 技術的アプローチ

- Custom GPT / Claude Projectsでキャラクター人格を作り込む
- glasswerksの知識・実績を完全に学習させる
- 音声は「Ho Hoo」の単一ファイルのみ（軽量実装）
- 任天堂的思想：「枯れた技術の水平思考」— 最新技術の誇示ではなくUX優先
- プラットフォーム規約を遵守（完全自動bot化は避ける）

---

## 7. まとめ

Hooは単なるマスコットキャラクターではなく、glasswerksの事業戦略における重要なピースである。

「仕組みは人間が作り、運用はAIが担う」という基本方針のもと、代表者の特性（創作への特化）を最大限に活かしながら、苦手領域（営業・定型対応）をAIで補完する構造を実現する。

まずはMUEDnoteのUIキャラクターとして導入し、ユーザーの反応を見ながら段階的に役割を拡張していく。
