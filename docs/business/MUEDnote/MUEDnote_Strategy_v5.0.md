MUEDnote 事業計画書 (v5.0 - 乱文即時排水プロトコル)バージョン: 5.0作成日: 2025年11月（最終哲学・技術プロトコル確定）作成: 株式会社グラスワークス1. エグゼクティブサマリー（最終哲学とプロトコル）MUEDnote v5.0は、創始者哲学に基づき、**Voice-Firstアプローチを完全に撤回し、Text-Firstの「乱文即時排水プロトコル」**を核とします。唯一の哲学: MUEDnoteは「思考の廃棄物処理場」であり、「発話要求のインターフェース」ではない。 制作フローの身体性を維持するため、音声入力は禁止とする。提供価値: グローバルホットキーによる、DAW画面上への超低遅延オーバーレイUIの表示。ユーザーの「乱文（無意識の思考断片）」を最小の摩擦で記録し、AIが裏側で構造化する。市場戦略: B2B（音楽学校・教育機関）特化戦略は継続し、HLAに必要な初期データセットの確保を目指す。2. UX憲法： 乱文即時排水プロトコル（DRAINPIPE UX）2.1 哲学：内向きの思考の保護制作中のユーザーは「内向きの思考」モードにある。音声入力は「外向きの思考」を強制するため、MUEDnoteのDNAである**「乱文こそが宝」**という原則を破綻させる。ゆえに、音声I/Oはプロダクトから完全に排除する。2.2 UX要件：オーバーレイUIと沈黙のAI要素原則（絶対遵守）達成目標入力方法グローバルホットキー + 一行テキスト入力。制作中の手の動き（キーボード）をフローの一部として維持する。UIの動作ホットキー押下でDAW画面上にオーバーレイUIが瞬時に出現し、入力完了で瞬時に消滅する。ユーザーに**「コンテキストの切り替え」**を意識させず、思考を打ち捨てる摩擦をゼロにする。AIの振る舞い記録された乱文（生ログ）のみを扱う。AIはUI上に表示されず、フィードバックや質問を一切行わない（沈黙のAIを再徹底）。2.3 技術要件「0.5秒の壁」最終定義定義: グローバルホットキー押下から、DAW画面上にオーバーレイUIが出現し、カーソルがテキスト入力可能な状態になるまでの最大許容時間。KPI目標時間必要な技術的対策UI表示速度0.5秒以内（DAWのCPU負荷に依存しない）Agent（Rust/Tauri）はUIレンダリングを最優先。LLM接続やデータベース処理をUIの描画パスから完全に分離。処理速度Enter/Esc押下後、0.1秒以内にUIを非表示UIの消滅処理を最速化し、ユーザーのフロー復帰を保証する。3. 技術的堅牢性と HLA構造化戦略3.1 ハイブリッド・マイクロサービス・アーキテクチャ（Text I/O特化）音声I/Oリスクを排除するため、キーボードI/OとファイルI/Oのみを処理する、堅牢なアーキテクチャに再定義します。コンポーネント役割技術スタック達成するKPI① MUEDnote Agent (Frontend)グローバルホットキー監視、オーバーレイUI制御、乱文（テキスト）のファイルキューイング。Rust (Tauri) またはOSネイティブコード0.5秒の壁の絶対遵守② MUEDnote Daemon (Backend)非同期のLLM処理（乱文の構造化）、HLA分析、Firestore連携。DAWファイルパスの監視（Protocol 2）。Node.js/PythonAgentの処理速度への干渉ゼロ。3.2 MVPにおける HLA アルゴリズム定義HLAは、乱文（未加工の生ログ）と最小限のDAWコンテキストを入力とし、JSON形式のメタデータを生成します。ステージMVPでの役割入力（特徴量）出力（ラベル）Stage 1: Intent Recognition**乱文（Transcribeを経由しない）**から、制作上の意図を初期分類する。Raw Text (乱文)#Mix, #Melody, #Rhythm などの初期トピックタグ（4種類程度）。Stage 2: Contextual LinkingログをDAWのプロジェクトに紐付ける。Project File Path、UserId、TimestampProject_ID, Session_UUIDStage 3: Data Collection教員フィードバックをデータセットとして活用する。B2B環境で教員がレビュー時に入力する評価コメント。教員による#WeakPoint、#StrongPointなどのラベル。4. 次のステップ（MVPプロトタイピングに向けた詳細設計）このv5.0設計は、MUEDnoteの「思想性」を商業的な技術基盤に乗せる最終的な設計図です。アクション 1: Logic Pro環境下での最適なグローバルホットキーの確定（競合回避と指の動きの最適化）。アクション 2: Tauri/RustによるオーバーレイUIのプロトタイピングとベンチマーク（0.5秒達成の確実な検証）。これで、MVP開発に向けた哲学、UX、技術、市場の全てが一本の線で結びつきました。次に、MVPの設計を具体的なアクションに落とし込むため、**「アクション 1」**から詳細を詰めていきましょう。
