## 1. エグゼクティブサマリー

### 1.1 プロダクト定義
MUEDnote（ミュードノート）は、音楽制作者（DAWユーザー）のための**「認知負荷の外部化インフラ」**である。

制作中のフロー状態を一切途切れさせないまま、乱文（思考断片）を記録・構造化し、知的資産に変換するシステム。

### 1.2 設計思想
- 表のUXは「完全沈黙」— 入力バーだけが一瞬現れ、返事はしない
- 裏ではAIが常時稼働し、構造化・HLA処理を全自動で行う
- MVPはDAW連携ゼロで出す（最速で価値提供）
- 製品版ではAU/VST/AAXによる完全DAW統合を実現する

### 1.3 解決する課題（The Pain）
| 課題 | 影響 |
|------|------|
| **コンテキストスイッチ** | メモやAI対話のためにDAWから離れると、フロー復帰に平均20分を要する |
| **短期記憶の限界** | 「後で直す」「今の判断の理由」といった思考が、記録されずに揮発する |
| **暗黙知の散逸** | 微細な判断がブラックボックス化し、再現性のある資産（IP）にならない |

### 1.4 ソリューション
**「DAWオーバーレイ型・沈黙のコンソール」**を提供する。

ユーザーは、0.5秒で出現する入力バーに「乱文」を投げ込むだけ。AIは「会話」を行わず、裏側で静かに文脈を解析し、将来の検索やドキュメント生成のためにデータを構造化（HLA）する。

## 2. UX憲法：乱文即時排水プロトコル（DRAINPIPE UX）

MUEDnoteの競争優位性は、機能ではなくこの「哲学」の遵守にある。

### 2.1 哲学：内向きの思考の保護
制作中のユーザーは「内向きの思考」モードにある。音声入力や対話型AIは「外向きの思考」を強制し、フローを破壊する。

ゆえに、**「入力はキーボードのみ」「AIは沈黙する」**ことを絶対原則とする。

### 2.2 技術的絶対要件：0.5秒の壁
| 要件 | 基準 |
|------|------|
| **出現** | ホットキー押下から入力可能まで **0.5秒以内** |
| **消滅** | Enter押下後 **0.1秒以内** にUI消滅 |
| **復帰** | フォーカスを即座にDAWへ戻す |

DAWが高負荷状態でも遵守する。

### 2.3 沈黙のAI（Silent Console）
- AIはUI上に表示されず、フィードバックや質問を一切行わない
- 視覚的な承認は「極小の点滅ドット」のみを許容
- ただし裏では以下を実行：
  - 乱文解析・文脈推定
  - HLA処理
  - 要約の蓄積
  - 自動メタデータ生成

### 2.4 入力UI仕様
- グローバルホットキー：`Cmd + Shift + Space`（例）
- 1行テキストバーのみ
- 入力した瞬間にDAWの世界へ戻れる身体感覚を提供

## 3. Human Learning Algorithm（HLA）

「ゴミ（乱文）」を「資産」に変える独自のデータパイプライン。

### 3.1 目的
人間の制作行動には、明文化されていない「癖・意図・判断基準・ストレスパターン」が存在する。HLAはそれを解析し、以下に変換する：

- 意図（何をしようとしていたか）
- 行動の特徴量（口癖、専門語反復、負荷指標）
- 感情傾向（ネガティブスパイラル検知）
- スランプ予兆
- 成功パターンの抽出

### 3.2 4段階プロセス

| Stage | 処理内容 | 出力データ例 |
|-------|----------|--------------|
| **1. Intent Recognition** | 乱文から制作上の意図を初期分類 | `Intent: #Mix`, `Target: #Bass` |
| **2. Context Integration** | DAWのプロジェクト名やファイルパスを自動取得し紐付け | `ProjectID: Song_A`, `Session: late_night` |
| **3. Feature Extraction** | 感情、行動パターン（癖）、スランプ傾向を数値化 | `Mood: Frustrated`, `Pattern: Loop_Fixation` |
| **4. Asset Generation** | 蓄積データを再構成し、ライナーノーツや自己分析レポートを生成 | `JSON Metadata`, `Liner Notes Draft` |

すべて裏で動作し、UIには一切出ない。

## 4. 技術アーキテクチャ（Silent AI Architecture）

「0.5秒の壁」を守るため、UI処理とAI処理を完全に分離する。

### 4.1 4層構造

| レイヤ | 技術 | 役割 |
|--------|------|------|
| **Agent** | Rust / Tauri | ホットキー監視、UI描画、ローカルキュー保存、DAWフォーカス管理 |
| **Daemon** | Node.js / Python | 非同期処理、AIへの送信、HLA処理、DB同期 |
| **AIエンジン** | GPT-5-mini / GPT-5-nano | 高速構造化、JSON mode |
| **Storage** | Firestore / Supabase | ログ永続化、パターンDB |

### 4.2 データ構造
- Raw Log（乱文）
- Structured Log（HLAによる構造）
- Summary Log（AI要約）
- Contextual Metadata（DAW情報）
- Pattern DB（個人特徴）

## 5. DAW連携戦略

### 5.1 Phase 1：MVP（DAW連携なし）
乱文の価値だけで勝負。「外部脳」としての習慣形成が目的。

### 5.2 Phase 2以降：4プロトコル

| Protocol | 手法 | 取得情報 |
|----------|------|----------|
| **1. Window Title Monitoring** | ウィンドウタイトル監視 | プロジェクト名 |
| **2. File Path Tracking** | プロジェクトファイル更新監視 | .als, .logicx等の状態 |
| **3. MIDI / OSC** | 間接取得（任意） | 再生/停止、アクティブトラック |
| **4. Keystroke Context Linking** | 直前操作の推定 | Cmd+Z → Undo系ログ等 |

公式APIに依存しない堅牢なコンテキスト取得戦略。
最終的にはAU/VST/AAXプラグインで**「DAWの内側に溶け込む外部脳」**を実現。

## 6. コア機能

### 6.1 Fragment Logging（乱文入力）
0.5秒で入力、沈黙で裏処理へ送信。

### 6.2 Auto Structuring（裏側自動構造化）
- タグ付け
- 意図解析
- 感情指標
- 過去ログとの関連付け
- DAW文脈（製品版）

### 6.3 Smart Search（検索）
蓄積した乱文を即座に検索。「あの時なんて書いたっけ」を0.5秒で解決。

### 6.4 Asset Generator（資産化機能）
- **Auto Liner Notes**: 制作過程のストーリー化
- **Smart Recall**: 過去の成功パターン再利用
- **Project Time Travel**: 数ヶ月前の判断の復元

## 7. ビジネスモデルと市場戦略

### 7.1 ターゲット市場

| セグメント | 対象 | 訴求ポイント |
|------------|------|--------------|
| **Primary（B2C）** | プロ・セミプロ・ハイアマ | 時間短縮・再現性・制作効率 |
| **Secondary（B2B）** | 音楽教育機関 | 講師の評価コスト削減（50%）、学習データ確保 |

### 7.2 競争優位性（Moat）
- **入力速度**: 汎用AI（ChatGPT等）には不可能な「0.5秒オーバーレイ」体験
- **独自データ**: DAW操作ログではなく「ユーザーの内面的な意図」が蓄積される唯一のDB
- **外部脳としての永続性**: 一度使い始めると離れられない依存構造

## 11. Exit戦略

### 11.1 想定買収先
- **DAWメーカー**: Ableton / Apple / Steinberg
- **クリエイターツール企業**: Spotify / Google / Adobe

### 11.2 資産価値
MUEDnoteの本質的価値は「制作意図のデータベース」にある。

これはDAWが将来AI化する際の**「ブラックボックスの鍵」**となる。AI作曲・AI編曲が進化しても、「人間がなぜその判断をしたか」のデータは、MUEDnoteにしか存在しない。

MUEDnote v6.1は、**沈黙・0.5秒・乱文・HLA**を軸に、「制作中の思考をそのまま知的資産に変える唯一のシステム」として設計を完了した。

次フェーズでは、本設計に基づきMVPの具体仕様（API、UIフロー、ストレージ設計）へ進む。
