# MUED Vision Evolution

MUEDのビジョン進化と今後の方向性についての議論ドキュメント。

---

## 1. 出発点（従来のLMSモデル）

```
講師（雇用）→ 受講者（課金）
```

- 運営がレッスンを提供
- 講師を集め、受講者から徴収
- 一般的なB2C型LMS

**課題:**
- 講師が集まらない
- 集客コストが高い
- 運営側にレッスン提供の意志がない

---

## 2. 転換点：MUEDnoteの開発

LMS待機中に、関連アプリで認知拡大を狙う：

- **MUEDnote**: 練習中の判断を音声で記録 → Whisperで文字起こし
- 哲学策定: 「出力はAI、判断と欲は人間」

**気づき:**
> 「あれ、これますます普通のLMSじゃない」

フィロソフィーを詰めるほど、従来のLMSから離れていった。

---

## 3. 新しい方向性：C2C相互メンタリング

### 3.1 コンセプト

```
クリエイター ⇄ クリエイター
「判断を見せ合う」「プロセスを共有する」
```

- 固定的な「先生/生徒」ではない
- **経験値の違うクリエイター同士**が学び合う
- AIが出力する時代、人間の「選び方」に価値がある

### 3.2 メンター権利システム（構想）

```
[メンバー] ──成長──→ [メンター権利取得] ──→ [開講]
```

**メンター権利の獲得条件（検討中）:**
- 特定分野の習熟レベル
- 独自の理論・結論を持っている（博士課程的）
- 実績やアウトプットの蓄積

**メンターができること:**
- MUED内でコース/セッション開講
- 独自の料金設定
- 自分のナレッジを体系化・公開

### 3.3 プラットフォーム構造

```
┌─────────────────────────────────────────────────┐
│     MUED - 自学と相互補完のプラットフォーム       │
├─────────────────────────────────────────────────┤
│                                                 │
│  【自学ツール】           【相互メンタリング】    │
│  ・MUEDnote              ・メンター検索          │
│  ・イヤートレーニング     ・セッション予約        │
│  ・教材ライブラリ         ・コース受講           │
│  ・練習ログ              ・フィードバック交換    │
│                                                 │
├─────────────────────────────────────────────────┤
│  【ナレッジベース】                              │
│  ・メンバーの判断記録が蓄積                      │
│  ・独自理論のアーカイブ                          │
│  ・「なぜその選択をしたか」の集合知              │
└─────────────────────────────────────────────────┘
```

### 3.4 収益モデル（案）

| 収益源 | 説明 |
|-------|------|
| 基本サブスク | 全メンバーから最低限の月額 |
| 取引手数料 | メンタリング発生時に利用料 |
| プレミアム機能 | 高度な分析、優先表示など |

## 4. 議論すべきポイント

### 4.1 メンター権利の設計

- 誰がメンターになれるのか？
- 審査制？自己申告制？実績ベース？
- 「独自の理論を持つ」をどう判定する？

### 4.2 差別化

- 既存のスキルシェア（ココナラ、ストアカ等）との違い
- なぜMUEDでなければならないか
- 「判断の記録」がどう差別化になるか

### 4.3 初期ユーザー獲得

- 鶏と卵問題（メンターもメンティーもいない）
- 最初のコアユーザーは誰か
- MUEDnoteからの導線

### 4.4 技術的検討

- メンター権利のシステム実装
- マッチングアルゴリズム
- 決済・手数料処理
- ナレッジベースの構造

## 5. 他AIとのディスカッション用質問
