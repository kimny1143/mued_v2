# 1. エグゼクティブサマリー

## MUEDnoteの概要

MUEDnote（ミュードノート）は、音楽制作のプロセスを可視化し、AI対話を通じて創作思考を構造化する次世代デジタルノートシステムです。

従来の制作ログツールが「何をしたか」の記録に留まっているのに対し、MUEDnoteはAIインタビュアーが「なぜそうしたか」「どう感じたか」を引き出し、制作者の暗黙知を形式知に変換します。これにより、音楽制作者の持続的な成長を支援いたします。

## 課題と解決

### 解決する課題

音楽制作者が直面する3つの主要な課題があります：

- **制作プロセスの振り返りが困難**: 制作時の思考や感情が記録されず、なぜその選択をしたのかが後から分からなくなる
- **上達の壁**: 自分の制作パターンや弱点が可視化されず、どこを改善すべきか分からない
- **講師とのコミュニケーションギャップ**: 制作過程での悩みや意図が正確に伝えられない

### MUEDnoteの解決策

**Session/Interview アーキテクチャ**により、制作作業を記録するだけでなく、AIが適切な質問を投げかけることで制作者の思考を引き出します。蓄積されたデータは、個人の成長分析や教材生成に活用されます。

## 市場性のポイント

- **対象市場**: 日本国内のDTM制作者200万人、音大生・専門学校生年間2万人
- **市場成長率**: 音楽教育SaaS市場は年率12.5%成長（2024-2030予測）
- **競合優位性**: 「制作プロセスの可視化 × AI対話」という独自領域で競合不在
- **収益モデル**: サブスクリプション型により早期マネタイズが可能

# 2. 製品概要

## 2.1 製品コンセプト

MUEDnoteは、音楽制作の各作業を「Session」という単位で管理し、AIインタビュアーが文脈に応じた質問を投げかけることで、制作者の思考プロセスを構造化します。

例えば、「サビのコード進行を調整した」というメモに対し、AIは「F→Gを入れた後、サビの入りは前より自然になりましたか？」といった具体的な質問を生成。制作者の回答は蓄積され、将来の制作や学習に活かされます。

## 2.2 プロダクトの動作

### 基本的な利用フロー

1. **Session作成**: 制作作業を開始する際、作業タイプ（作曲、練習、ミックスなど）を選択してSessionを開始
2. **短いメモの記録**: 何を行ったか簡単にメモを記録（例：「サビのコード進行を調整」）
3. **AI質問の生成**: AIがメモを分析し、2-3個の具体的な質問を生成
4. **回答と保存**: 制作者が質問に答えることで、思考プロセスが言語化され保存される
5. **振り返りと分析**: 蓄積されたSessionから自分の制作パターンや改善点を発見

## 2.3 技術の要点

MUEDnoteは最新のWeb技術とAI技術を組み合わせて構築されています：

- **AI対話エンジン**: OpenAI GPT-5系を活用し、音楽制作に特化した質問生成
- **RAG（Retrieval-Augmented Generation）**: 過去の制作ログを参照し、文脈に応じた質問を生成
- **セキュアな基盤**: 認証システムと暗号化により、制作データを安全に管理

# 6. 競合分析

## 直接競合

現時点で「音楽制作プロセスの可視化×AI対話」を提供する直接競合は存在しません。

## 間接競合

### 汎用ノートツール（Notion、Evernote）
- **強み**: 汎用性が高く、知名度がある
- **弱み**: 音楽制作に特化した機能なし、AI対話機能なし

### 音楽制作コラボツール（Splice）
- **強み**: サンプル共有、コラボレーション機能
- **弱み**: プロセス記録機能が弱い、思考整理機能なし

### AIマスタリングツール（Landr）
- **強み**: AI技術を活用した自動マスタリング
- **弱み**: 制作プロセスの記録機能なし、学習支援なし

## 差別化ポイント

MUEDnoteの独自性：

1. **Session/Interview構造**: 単なる記録でなく、AIが能動的に質問を投げかける
2. **音楽制作特化**: DTM制作に最適化されたワークフロー
3. **成長支援機能**: 蓄積データから弱点分析、改善提案を実施
4. **教育連携**: 講師・生徒間の共有機能により教育効果を最大化
