## エグゼクティブサマリー

MUEDnote（ミュードノート）は、**音楽制作のプロセスを可視化し、AI対話を通じて創作思考を構造化する**次世代デジタルノートシステムです。

### ミッション

**「非言語的な音楽制作プロセスを言語化し、学習と成長を加速する」**

従来の制作ログツールは「何をしたか」の記録に留まっていましたが、MUEDnoteはAIインタビュアーが**「なぜそうしたか」「どう感じたか」**を引き出し、暗黙知を形式知化します。

### 市場機会

- **音楽制作市場**: 世界で3億人以上（DTMer、音大生、プロ制作者）
- **成長率**: 音楽教育SaaS市場は年率12.5%成長（2024-2030予測）
- **痛点**: 制作プロセスの振り返りが困難、上達の壁、講師とのコミュニケーションギャップ

### 差別化要因

1. **Session/Interviewアーキテクチャ**: 単なるログではなく、AIが質問を投げて回答を構造化
2. **段階的実装戦略**: MVP（テキストベース）→ DAW統合 → MIDI/WAV解析
3. **RAG（Retrieval-Augmented Generation）**: 過去ログとテンプレートを活用した文脈依存質問生成
4. **ログデータ資産**: ユーザーの制作プロセスログは、将来の教材生成における最大の差別化要因

### 収益モデル

- **個人向けサブスクリプション**: ¥980/月（年払い¥9,800）
- **教育機関向けライセンス**: ¥50,000/年/講師（学生10名まで）
- **エンタープライズ**: ¥500,000〜/年（カスタマイズ、API提供）

### 目標

- **12ヶ月目**: 1,000ユーザー、MRR ¥980,000
- **24ヶ月目**: 10,000ユーザー、MRR ¥9,800,000
- **36ヶ月目**: 教育機関50校導入、エンタープライズ5社

## MUEDnoteの戦略的意義

### 1. 早期マネタイズ戦略

MUEDnoteは**Session/Interviewという革命的機能**により、既存市場にないソリューションを提供します。サブスクリプションモデルにより、ローンチ直後から収益化が可能です。

**他の音楽SaaSとの違い**:
- Notion/Evernote: 汎用ノートツール（音楽特化なし）
- Splice: サンプル共有・コラボ（プロセス記録なし）
- Landr: AIマスタリング（思考整理なし）

MUEDnoteは**「制作プロセスの可視化 × AI対話」**という独自領域で、競合不在の市場を開拓します。

### 2. データ資産の構築

ユーザーの制作ログ（Session/Interview Q&A）は、**将来の教材生成における最大の差別化要因**となる学習データです。

**データの価値**:
```
1,000ユーザー × 月10セッション × 3質問 = 月30,000のQ&Aペア
12ヶ月後 = 360,000のQ&Aペア（音楽制作プロセスの構造化データ）
```

このデータは：
- **弱点分析**: ユーザーが詰まるポイントの自動抽出
- **パーソナライズ教材生成**: 個別の学習パスの自動作成
- **質問テンプレート改善**: RAG検索品質の継続的向上

に活用され、**データネットワーク効果**（ユーザー増→教材品質向上→新規ユーザー獲得）を生み出します。

### 3. MUED全体のPoC（概念実証）

MUEDnoteは、**MUED全体の方向性を決定するPoC**として機能します。

**検証項目**:
- **ユーザー獲得**: 音楽制作者はAI対話型ツールを使うのか？
- **収益化**: サブスクリプションモデルは成立するのか？
- **ログデータの質**: 教材生成に使える構造化データは取れるのか？
- **機能の優先順位**: レッスン管理 vs 教材生成、どちらが求められるのか？

**MUEDnoteの成功が決めるもの**:
- ✅ 成功 → ログベース教材生成に注力、レッスン管理は後回し
- ❌ 不振 → 既存のレッスンマッチング路線に戻る

### 4. 独立製品としての展開

MUEDnoteは**独立製品**として展開し、既存のMUED LMS（レッスン管理システム）とは別にマーケティングを行います。

**理由**:
- **ターゲットが異なる**: MUEDnote（個人クリエイター）≠ MUED LMS（講師・生徒マッチング）
- **使用タイミングが異なる**: 制作直後の日常的使用 ≠ レッスン時の限定的使用
- **マネタイズモデルが異なる**: サブスク ≠ マッチング手数料

**将来的な統合（Phase 3以降）**:
- 講師が生徒のMUEDnoteログを閲覧してフィードバック
- ログから生成された教材をMUED LMSで販売
- ただし、MUEDnoteの成功次第で統合の要否を判断

## 2. 製品・サービス詳細

### 2.1 コア機能（MVP v0）

#### Session概念
制作作業を「Session」単位でグループ化：

```
Session = {
  type: "composition" | "practice" | "mix" | "ear_training" | "listening" | "theory",
  title: "サビのコード進行調整",
  userShortNote: "Amメロからサビ入りが唐突に感じたので、F→Gを挟んでみた",
  dawMeta: { tempo, key, bars },
  aiAnnotations: { focusArea: "harmony", intentHypothesis: "サビへの流れを滑らかにする意図" }
}
```

#### AIインタビュアー
1. **Analyzer**: userShortNoteを分析し、focusArea（harmony/melody/rhythm/mix）とintentHypothesisを推定
2. **RAG検索**: 過去のQ&A + 質問テンプレートから文脈関連情報を取得
3. **Interviewer LLM**: 2〜3問の質問を生成
   - 「Amメロ→サビの境界で、耳で聴いた時に"あ、このタイミングだな"と感じた瞬間はどこ？」
   - 「F→Gを入れた後、サビの入りは"前より自然/まだ唐突"どっち？」
4. **回答保存**: ユーザーの回答をRAGに蓄積し、次回の質問生成に活用

#### データフロー

```
User → Short Note
  ↓
Analyzer (text-based, MVP)
  ↓
SessionAnalysis { focusArea, intentHypothesis }
  ↓
RAG Retrieval (past logs + templates)
  ↓
Interviewer LLM → Questions (2-3)
  ↓
User Answers → RAG Storage
```

#### Frontend
- **Next.js 15.5.4** (App Router, React Server Components)
- **React 19** (useOptimistic, streaming)
- **TailwindCSS 4** (design system)
- **Shadcn/UI** (component library)

#### Backend
- **Next.js API Routes**
- **Drizzle ORM** (type-safe SQL queries)
- **Clerk Authentication** (RLS統合)
- **Neon PostgreSQL** (serverless, pgvector拡張)

#### AI
- **OpenAI GPT-5系** (本番環境、gpt-5-mini推奨)
- **Claude Sonnet 4.5** (開発・管理者モード、MCP Server経由)
- **OpenAI Embeddings** (text-embedding-ada-002, RAG用)

#### Infrastructure
- **Vercel** (deployment, edge functions)
- **GitHub Actions** (CI/CD, Playwright E2E)

### 2.3 データベーススキーマ

#### Core Tables
1. **sessions** - Session情報（type, title, userShortNote, dawMeta, aiAnnotations）
2. **session_analyses** - Analyzerの出力（focusArea, intentHypothesis, confidence）
3. **interview_questions** - 生成された質問（text, focus, depth, order）
4. **interview_answers** - ユーザーの回答（text, aiInsights）
5. **rag_embeddings** - RAG用ベクトルストア（pgvector）

#### RLS（Row Level Security）
- Clerk認証と統合
- ユーザー自身のセッションのみアクセス可能
- shareWithMentor = true の場合はメンター閲覧可能

### 3.2 競合分析

| 製品/サービス | 強み | 弱み | MUEDnoteの差別化 |
|--------------|------|------|------------------|

**結論**: 「制作プロセスの可視化 × AI対話」の組み合わせは他社にない独自性

## 8. リスクと対策

### 8.1 技術リスク

| リスク | 影響 | 確率 | 対策 |
|--------|------|------|------|
| **MIDI/WAV解析精度が低い** | 高 | 中 | MVP版テキストベースで先行リリース、段階的移行 |
| **RAG検索品質が低い** | 中 | 中 | 質問テンプレート手動キュレーション、embeddings最適化 |
| **スケーラビリティ問題** | 高 | 低 | Neon Serverless PostgreSQL、Vercel edge functions活用 |
| **AI API コスト爆発** | 高 | 中 | キャッシング戦略、gpt-5-mini使用、Rate limiting |

### 8.2 ビジネスリスク

| リスク | 影響 | 確率 | 対策 |
|--------|------|------|------|
| **ユーザー獲得が遅い** | 高 | 中 | Free Tierでの裾野拡大、インフルエンサー施策 |
| **Pro転換率が低い** | 高 | 中 | オンボーディング改善、価値提示の強化 |
| **教育機関導入が進まない** | 中 | 中 | トライアル無償提供、導入事例作成 |
| **競合参入** | 中 | 低 | 特許申請、先行者優位の確立（データネットワーク効果） |

### 8.3 規制・法務リスク

| リスク | 影響 | 確率 | 対策 |
|--------|------|------|------|
| **著作権問題（生成教材）** | 高 | 低 | 利用規約での免責、教材は「提案」として提示 |
| **個人情報保護** | 高 | 低 | ISO 27001準拠、RLS実装、暗号化 |
| **AI倫理問題** | 中 | 低 | 透明性の確保、ユーザー同意取得 |

## 12. 結論

MUEDnoteは、**音楽制作プロセスの可視化**という未開拓市場に対し、**Session/Interviewアーキテクチャ**と**RAG技術**を組み合わせた独自ソリューションを提供します。

### 戦略的意義

1. **早期マネタイズ**: 競合不在の市場で即座に収益化
2. **データ資産構築**: ログデータは将来の教材生成における最大の差別化要因
3. **MUED全体のPoC**: MUEDnoteの成功が、MUED全体の方向性を決定

### 成功の鍵

1. **MVP速度**: テキストベースで早期リリース、フィードバック収集
2. **ログデータ品質**: ユーザーが日常的に記録し、AI質問に回答する仕組み
3. **データネットワーク効果**: 早期ユーザー獲得によるRAG品質向上→新規ユーザー獲得
4. **段階的実装**: 技術リスクを分散し、継続的価値提供
5. **PoCとしての機能**: 24ヶ月後の戦略決定（ログベース教材生成 vs レッスン管理）

### 最終ゴール

**Year 2終了時点での判断**:
- ✅ MUEDnote成功 → ログベース教材生成に全力投球、レッスン管理は後回し
- △ 中程度の成功 → 選択的にMUED LMS統合
- ❌ 不振 → 既存レッスン路線に戻る

**この柔軟な戦略により、リスクを最小化しながら最大のリターンを目指す**
