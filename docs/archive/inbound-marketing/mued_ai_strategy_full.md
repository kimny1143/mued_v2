# MUED AI集客戦略・アーキテクチャ設計書（完全版）

## はじめに
本書は、MUEDが想定する主要ターゲット（初心者・独学クリエイター・海外ユーザー）を対象にした、**AI主導の集客戦略と技術アーキテクチャの統合設計**をまとめたものです。

MUEDの核は次の三点です。

- **AI診断による自動集客**
- **無限増殖する小粒教材（AI＋MUEDnote生成）**
- **バイリンガル教材による世界展開**

これを支えるエコシステム全体を構造化し、実装レベルまで落とし込んでいます。

---

# 1. MUEDのターゲット戦略

## 1.1 対象ユーザー
MUEDは「潜在顧客のまま収益化できる」エデュテックを目指すため、以下の層に特化する。

1. **独学クリエイター（日本／海外）**  
   - 自分の弱点がわからない  
   - 評価基準が曖昧  
   - 学習意欲は高い

2. **初心者**  
   - 階段を一段ずつ登れる教材を求める  
   - 自己判断ができない

3. **海外クリエイター（英語圏）**  
   - J-PopやAnime系の作曲に興味  
   - 良質な教材が不足  
   - 言語壁をAIで吸収できる

## 1.2 MUEDのコンセプト
MUEDは「教材体系の教育サービス」ではなく、

**制作ログ × AI × 小粒教材 × SNS拡散  
で成長する“音楽ナレッジOS”**

として設計する。

---

# 2. 集客戦略（AI主導の導線設計）

## 2.1 集客の核心
MUEDの入口は「ユーザー作品」そのものが担う。

作品診断AIが  
- 魅力の言語化  
- 改善点  
- キャッチコピー  
- SNS投稿文（日英）  
を自動生成し、  
**シェアされるための“物語”を付与する。**

これがMUEDの自動集客エンジン。

## 2.2 集客導線の全体像
1. **AI作品診断（無料）**  
2. SNS拡散（英語／日本語）  
3. MUEDにアクセス  
4. 分析レポート表示  
5. 教材レコメンド（小粒教材）  
6. パッケージ生成（AI自動編成）  
7. 購入  
8. MUEDサブスク利用（習熟マップ）  
9. MUEDnoteで制作ログ蓄積  
10. ログから新教材が生成  
11. 再度レコメンド  
12. 継続課金

**“集客→収益→教材増殖→再集客”の永久ループ** が成立する。

---

# 3. 教材戦略

## 3.1 小粒教材（細分化モデル）
教材は可能な限り細かく分割して提供する。

- 1教材＝1技法  
- 価格は低単価  
- 難易度・スキル・弱点タグをメタデータ化  
- テキスト／図解／音声例／短尺動画 の混在を許可

## 3.2 無限増殖型教材の生成原則
教材の増加源は3つ。

1. **MUEDnoteの制作ログ**  
2. **あなたの既存ナレッジの細分化**  
3. **AIの文章再構成**

教材は体系に固定されず、図書館のように増殖する。

## 3.3 パッケージ生成（AI）
ユーザーの  
- 弱点  
- 目的  
- ジャンル嗜好  
- 現在のスキル  
を基に、AIが教材を束ねた「指向パッケージ」を作る。

# 4. 技術アーキテクチャ

## 4.1 全体構成
- **Frontend:** Next.js or Nuxt  
- **Backend:** Python（Flask）または Node（Express）  
- **DB:** Supabase（PostgreSQL）  
- **File Storage:** Supabase Storage / GCS  
- **AI:** OpenAI GPT + Embeddings  
- **音響解析:** Essentia / librosa / ffmpeg  
- **配信:** Vercel / Railway  

## 4.2 教材データモデル（バイリンガル対応）
教材ID  
- metadata（言語非依存）  
- content_ja（テキスト本体）  
- content_en（テキスト本体）  
- media（画像・音声・動画）  
- tags  
- 弱点タグ  
- 関連教材ID  
- 生成元（MUEDnote / AI / 手動）  

## 4.3 AI診断フロー
1. 音源／MIDI／鼻歌の入力  
2. 特徴量抽出（Key・Chord・BPM・構造 etc）  
3. LLMに情報を渡して  
   - 魅力  
   - 弱点  
   - スキル分析  
を生成  
4. SNS投稿文（日英）出力  
5. 教材レコメンドへ  

## 4.4 翻訳・同期
バイリンガル教材は必ず  
- content_ja → GPTで構造解析  
- content_en → GPTで整合性チェック  
を行い、一貫性を保つ。

# 5. 収益モデル

## 5.1 ハイブリッドモデル（A+B）
1. **AI診断 → MUEDサブスク（月額）**  
   -習熟マップ  
   -ログ分析  
   -定期診断

2. **AI診断 → 教材単品販売（都度課金）**  
   -改善点に直結した教材を購入  
   -購入の心理的ハードルが低い  

## 5.2 追加収益
- MUEDnoteアプリ  
- パッケージ販売  
- 海外向け教材の拡張

として半自動登録。
