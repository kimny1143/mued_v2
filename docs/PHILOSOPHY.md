# MUED Philosophy - 音楽制作特化プラットフォームの思想

**Version**: 1.0 (Draft)
**Created**: 2025-11-15
**Status**: Phase 0 Foundation Document

## 1. Core Philosophy - 3つの柱

MUEDは音楽制作における人間の成長を「**Difference（耳）**」「**Note（制作・学習ログ）**」「**Form（構造）**」の3つの観点から支援する、音楽制作特化プラットフォームです。

### 1.1 Difference（差分を聴く耳）

**定義**: 音の微細な違いを聴き分け、判断する能力

**具体例**:
- EQの1dBの違いを認識する
- ミックスバランスの変化を感じ取る
- リズムの微細なずれを検出する
- 音色の質感の違いを言語化する

**システムでの実装**:
- `EarExercise`モデル: A/B比較問題
- 差分メタデータ管理
- 正答率・成長曲線の可視化
- 主観的な気づきの記録（LogEntryとの連携）

### 1.2 Note（制作・学習ログ）

**定義**: あらゆる音楽活動における判断・気づき・成長の記録

**重要性**: MUEDの最上位概念。すべての活動がログとして資産化される

**記録対象**:
- レッスンでの学び
- 練習中の気づき
- 制作時の判断理由
- AIとの対話履歴
- 耳トレーニングの結果と感想

**システムでの実装**:
- `LogEntry`モデル: 統一的なログ管理
- AI要約機能: 散発的な記録を体系化
- タイムライン表示: 成長の軌跡を可視化
- タグ・感情・難易度によるフィルタリング

### 1.3 Form（構造を理解する力）

**定義**: 楽曲の構造・展開・機能を分析し、制作に活かす能力

**要素**:
- セクション構成の把握
- コード進行の機能理解
- アレンジパターンの認識
- 展開の必然性の理解

**システムでの実装**:
- `FormExercise`モデル: 構造分析問題
- 波形・ピアノロール可視化
- AI構造解析エンジン
- 主観的解釈の記録（LogEntryとの連携）

## 2. 統合的な学習循環

### 2.1 3つの柱の相互作用

```
Difference（耳）
    ↓ 聴き取った違いを
Note（ログ）← 記録・言語化
    ↓ 蓄積された知見から
Form（構造）← パターンを発見
    ↓ 構造理解が深まると
Difference（耳）← より精密な聴き分けが可能に
```

### 2.2 AI機能との連携

**AI教材生成**:
- Differenceレベルに応じた問題難易度調整
- Form理解度に基づいた構造例題の提供
- Note履歴を参照した個別最適化

**AI要約・分析**:
- 散発的なNoteを体系的知識に変換
- Differenceトレーニングの弱点分析
- Form学習の進捗パターン抽出

**AIメンタリング**:
- Note履歴に基づいた学習アドバイス
- Difference能力の成長曲線予測
- Form理解の次ステップ提案

## 3. 既存機能との関係性

### 3.1 教材（Materials）

**拡張方向**:
- 従来: 楽譜・音源の提供
- 新規: Difference/Form特化型教材の追加
- 統合: すべての教材利用がNoteに記録される

### 3.2 レッスン（Lessons）

**拡張方向**:
- 従来: 予約・実施・フィードバック
- 新規: レッスン内容のNote自動生成
- 統合: Difference/Form能力の成長追跡

### 3.3 ダッシュボード

**拡張方向**:
- 従来: 予約状況・学習進捗
- 新規: 3つの柱での成長可視化
- 統合: Note履歴からのインサイト表示

## 4. 実装優先順位（Phase別）

- クローズドβリリース

## 5. 成功指標

### 定量的指標
- Note記録率: 全活動の80%以上
- Difference正答率向上: 3ヶ月で20%改善
- Form理解度テスト: 平均70点以上

### 定性的指標
- ユーザーが自身の成長を実感できる
- 音楽制作の判断理由を言語化できる
- 他の学習者と知見を共有したくなる

## 6. 技術的制約と対応

### 制約事項
1. **データプライバシー**: Noteは個人の創造的資産
2. **計算リソース**: AI分析のコスト管理
3. **UX複雑性**: 3つの柱の統合的な表現

### 対応方針
1. **プライバシー**: デフォルト非公開、段階的共有機能
2. **リソース**: オンデマンド分析、キャッシュ活用
3. **UX**: 段階的な機能開放、オンボーディング強化

## 7. 長期ビジョン

### 5年後の姿
- 音楽制作者の「第二の脳」として機能
- 個人の成長データが最大の資産に
- コミュニティ全体の集合知が形成される

### 10年後の姿
- 音楽教育の新しいスタンダード
- AIと人間の共創が当たり前に
- 創造プロセスの科学的解明に貢献
